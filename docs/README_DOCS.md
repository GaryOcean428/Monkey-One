All documentation should follow:

1. **ISO-aligned naming**: Use kebab-case for filenames
2. **Clear structure**: Use headers, sections, and lists
3. **Code examples**: Include practical examples where relevant
4. **Up-to-date**: Keep documentation synchronized with code changes
5. **Comprehensive**: Cover edge cases and common issues
